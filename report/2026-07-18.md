### 1. Token Verification Middleware
* Created a security utility at `backend/utils/verifyUser.js`.
* Implemented `verifyToken` middleware to extract the JWT `access_token` from client cookies, verify it using the `JWT_SECRET` key, and attach user information to `req.user`.

### 2. User Update Route & Controller Logic
* Created the `POST /api/user/update/:id` API route mapping in `backend/routes/user.route.js`, protecting it with the new `verifyToken` authentication guard.
* Implemented the `updateUser` controller logic in `backend/controllers/user.controller.js`:
  * Validates that the logged-in user matches the target profile parameter (`req.user.id === req.params.id`).
  * Hashes passwords securely using `bcryptjs` if a password update is requested.
  * Updates properties (`username`, `email`, `password`, `avatar`) in MongoDB using `findByIdAndUpdate`.
  * Destructures the updated document to exclude the password hash before sending the remaining profile data back to the client in a JSON payload.

### 3. Server Configuration & Dependency Updates
* Installed `cookie-parser` as a core backend dependency to parse cookie request headers.
* Registered the cookie-parser middleware (`app.use(cookieParser())`) in `backend/index.js` before router paths.

### 4. Redux State & Profile Integration
* Updated `client/src/redux/user/userSlice.js` with `updateUserStart`, `updateUserSuccess`, and `updateUserFailure` actions and state reducers.
* Connected `Profile.jsx` to dispatch update actions and send `POST` requests to `/api/user/update/:id`.

### 5. Delete User Controller Logic
* Implemented the `deleteUser` controller function in `backend/controllers/user.controller.js`.
* Added security authorization verification: compares `req.user.id` (extracted by `verifyToken` middleware) against `req.params.id`. If they do not match, returns a `401 Unauthorized` error (`You can only delete your own profile`).
* Interacted with MongoDB using `User.findByIdAndDelete(req.params.id)` to permanently remove the target user document.
* Returns a `200 OK` JSON response (`User deleted successfully`) upon completion.

### 6. Delete User Route Mapping
* Configured the `DELETE /api/user/delete/:id` endpoint in `backend/routes/user.route.js`.
* Protected the route using the `verifyToken` middleware guard.
