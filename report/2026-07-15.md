# Daily Work Report - July 15, 2026

## 🎯 Focus: Sign-In API & Error Handling

Today's focus was completing the core authentication endpoints on the backend with JSON Web Tokens (JWT) and introducing robust global error handling, alongside client-side code refactoring.

---

## 🛠️ Work Done

### 1. Sign-In Endpoints & Authentication (JWT)
* Created the `/api/auth/signin` API endpoint inside the auth controller.
* Validated credentials (user query and bcrypt password comparison).
* Generated JSON Web Tokens (JWT) using a custom secret key.
* Configured the server to return the token as an HTTP-only cookie, securing it from client-side script access.

### 2. Custom Error Handling Middleware
* Created `backend/utils/error.js` containing helper utility functions (e.g., `errorHandler`) to standardise HTTP error responses.
* Configured a global Express error-handling middleware inside `backend/index.js` to catch asynchronous failures and return neat JSON errors to the client.

### 3. Client Refactoring & Auth Views
* Created a dedicated Auth directory: `client/src/pages/Auth/`.
* Created new components: `SignIn.jsx` and `SignUp.jsx`.
* Cleaned up and deleted legacy files (`Login.jsx` and `Register.jsx` from `Customer` folder).

### 4. Git Hygiene
