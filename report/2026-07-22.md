# Daily Work Report - July 22, 2026

## 🎯 Focus: Sign-Out Implementation & Create Listing API Route

Today's focus was adding a secure sign-out mechanism for users to clear their session cookies and building the database structure and backend routes to support property listing creation.

---

## 🛠️ Work Done

### 1. User Sign-Out Integration
* **Backend:**
  * Implemented the `signOut` controller logic inside `backend/controllers/auth.controller.js` to clear the `access_token` cookie from the client's browser.
  * Added the `/api/auth/signout` route in `backend/routes/auth.route.js`.
* **Frontend:**
  * Added `signOutUserStart`, `signOutUserSuccess`, and `signOutUserFailure` action reducers in `client/src/redux/user/userSlice.js`.
  * Connected the "Sign out" label in `client/src/pages/Customer/Profile.jsx` to fetch the `/api/auth/signout` route, clear the Redux user state, and redirect to the login/home screen.

### 2. Create Listing API Route
* **Database Modeling:**
  * Designed the listing Mongoose Schema in `backend/models/listing.model.js` with fields for name, description, address, regularPrice, discountPrice, bathrooms, bedrooms, furnished status, parking availability, listing type (rent/sell), offer status, imageUrls array, and reference to the owner (`userRef`).
* **Router & Controller Configuration:**
  * Created `backend/controllers/listing.controller.js` to handle saving listing metadata in MongoDB.
  * Setup `backend/routes/listing.route.js` with `POST /create` protected by the `verifyToken` security guard.
  * Configured backend entry point `backend/index.js` to register `listingRouter` at `/api/listing`.
